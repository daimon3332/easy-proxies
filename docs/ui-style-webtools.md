# WebTools UI 风格说明

## 整体风格

WebTools 整体是浅色蓝绿系工具站风格，视觉气质偏清爽、轻量、现代，强调快速使用和低干扰操作。界面不追求厚重后台感，而是更接近在线开发工具、配置编辑器、轻量 SaaS 工具台的感觉。

页面基底以浅色为主，整体明亮干净。背景不使用大面积深色或沉重灰色，而是通过浅蓝、浅青、白色之间的柔和层次营造空间感。视觉上保持克制，避免过度装饰，让用户注意力集中在输入、编辑和结果区域。

主色体系以青绿色与天空蓝为核心。青绿色用于强调积极状态、成功反馈和部分激活状态；蓝色用于主导航、主要按钮、聚焦状态和当前页面高亮。整体色彩饱和度适中，偏柔和、明亮、轻盈，不是高对比霓虹风。

整体风格关键词可以概括为：浅色基底、蓝绿主色、清爽工具站、全宽工作台、左右双栏、轻量面板、柔和圆角、淡色边框、低干扰操作流、高可读编辑区、等宽字体、现代 SaaS 感、配置工具感、开发者工具感。

## 色彩体系

WebTools 的颜色体系克制、明亮、偏开发工具感。

主要色彩关系：

- 蓝色：用于主导航、主要按钮、聚焦状态、当前页面高亮。
- 青绿色：用于成功状态、激活状态、正向反馈。
- 白色：用于输入区、输出区、工具面板主体。
- 浅蓝灰：用于页面背景、分隔线、弱面板背景。
- 灰蓝色：用于说明文字、占位文本、次级信息。
- 浅红色：用于错误状态和危险提示。

颜色使用重点是低干扰。蓝色和绿色只在需要表达当前状态、主要动作、成功结果时出现，不大面积铺满页面。页面整体不追求强对比，而是通过轻微色彩层次建立空间感。

## 总体排版

WebTools 采用全宽工作台式排版。页面充分利用横向空间，尤其在桌面端形成明显的左右双栏结构：左侧用于输入，右侧用于输出或预览。整体操作路径清晰，符合“从左到右完成处理”的工具型使用习惯。

页面不应该被限制在很窄的居中容器中。输入输出类工具需要横向空间，因此主体区域应尽量占满浏览器宽度，只保留适度外边距。

整体页面结构通常是：

- 顶部导航。
- 工具标题区。
- 操作按钮区。
- 状态提示区。
- 输入输出工作区。

排版密度介于“宽松展示型”和“高密后台型”之间。控件间距紧凑但不拥挤，工具栏按钮排列密集但清晰，编辑区占据主要空间。界面明显服务于实际操作效率，而不是展示型视觉效果。

## 顶部导航

顶部导航是扁平工具栏设计。导航栏高度克制，不占用过多垂直空间，主要空间留给实际操作区。

顶部导航结构：

- 左侧是品牌标识。
- 右侧是横向工具入口。
- 当前页面使用蓝色高亮。
- 非当前项保持轻量文字按钮状态。
- Hover 时出现浅蓝背景反馈。

导航栏视觉特征：

- 白色或半透明白色背景。
- 底部使用浅色分隔线。
- 可以有轻微背景模糊。
- 导航项使用圆角按钮样式。
- 窄屏时允许横向滚动。

顶部导航只负责全局入口和当前页面识别，不应该占用过多高度，也不应该使用大标题、大横幅或重装饰。

## Logo 与品牌区

Logo 位于顶部导航左侧，是站点识别的视觉锚点。

Logo 风格：

- 圆角方形。
- 蓝色或蓝绿色背景。
- 白色字母标识，例如 W。
- 尺寸紧凑，与导航栏高度协调。
- 右侧显示 WebTools 品牌文字。

品牌区要清晰但克制。它不是 Hero 区，也不应该压缩工具主体空间。

## 工具主体布局

工具主体使用统一的面板式布局。每个工具页面顶部都有标题、简短说明和操作按钮区，下方是输入输出工作区。

标题区：

- 显示当前工具名称。
- 标题字号适中，不需要过大。
- 说明文字使用低饱和度灰蓝色。
- 标题区紧凑清晰，避免干扰主要操作。

操作按钮区：

- 靠近标题或位于标题右侧。
- 按钮集中排列。
- 支持自动换行。
- 不要把主要操作放到页面底部。

主体工作区：

- 输入输出区域占据最大空间。
- 工具栏和说明不应该挤压编辑区。
- 页面视觉重心始终在输入、输出、预览和编辑器。

## 面板与卡片风格

WebTools 的卡片和面板整体是浅边框、白底、轻分隔风格。边框颜色偏淡蓝灰，主要用于建立区域边界，而不是制造强烈视觉割裂。

面板视觉特征：

- 白色背景。
- 淡蓝灰细边框。
- 中等圆角。
- 阴影使用较少，整体偏平面化和轻量化。
- 面板头部只保留必要标签，例如“输入”“输出”。
- 面板内部强调编辑区域，不做过多装饰。

相比 Clash Template，WebTools 的玻璃感更弱，工具感更强，更接近纯白开发工具面板。

## 圆角体系

圆角是界面的重要视觉特征。导航项、按钮、输入框、编辑区域和状态提示都采用圆角处理，使界面更柔和、亲和，也更符合浅蓝绿色系的轻量工具风格。

圆角应统一：

- 顶部导航按钮：小圆角或胶囊圆角。
- 普通按钮：中等圆角。
- 输入输出面板：中等圆角。
- 状态提示：圆角提示条。
- Logo：较大圆角方形。

整体避免硬边、厚边框、直角后台风。

## 按钮风格

按钮层级清晰：

- 主操作按钮使用蓝色实底，视觉权重最高。
- 激活态按钮使用绿色强调色，表示当前查看模式或成功状态。
- 普通按钮使用白底、浅边框和灰蓝文字。
- Hover 状态通过蓝色边框、浅蓝背景或蓝色文字体现。
- 禁用状态降低透明度，减少误操作暗示。

WebTools 的按钮是工具栏按钮，不是大号营销按钮。它们应该紧凑、清晰、可快速点击。

按钮排布原则：

- 常用操作放前面。
- 危险操作放后面。
- 同类操作靠近。
- 操作按钮不要远离输入输出区域。

## 输入区与编辑区

输入区和编辑区采用代码编辑器式视觉。文本区域使用等宽字体，字号较小但可读性稳定，适合 YAML、JSON、命令行、URL、Base64 等配置型文本。

编辑器背景接近白色，边框和分隔线柔和，整体像专业配置工具而不是普通表单。

普通 textarea 风格：

- 等宽字体。
- 字号较小但清楚。
- 行高舒适。
- 背景简洁。
- 边框弱化。
- 输入区域尽量占满面板。

CodeMirror 编辑器风格：

- 保持开发者工具感。
- 行号区域使用浅灰背景。
- 正文使用等宽字体。
- 行高舒适。
- 聚焦时不出现刺眼边框。
- 语法高亮服务可读性，不要过度鲜艳。

普通 textarea 和 CodeMirror 应保持统一的等宽字体与简洁背景，使不同工具之间的输入体验一致。

## 分栏设计

分栏设计是当前 UI 的核心特征。

桌面端主要采用左右双栏：

- 输入和输出并排展示。
- 便于用户对照原始内容和处理结果。
- 默认比例接近 50/50。
- 输入输出区域高度一致。

部分短文本工具采用上下分栏：

- 输入在上。
- 输出在下。
- 更适合命令、短文本或短结果处理。
- 避免短内容被左右区域压缩。

分栏的核心目标是提高处理效率，让用户不用频繁滚动或切换页面就能完成输入和结果确认。

## 拖拽分隔条

分隔条很细，默认低存在感，hover 或拖拽时出现浅蓝反馈。

视觉要求：

- 不使用粗重分割线。
- 默认只承担结构提示。
- Hover 时增强可交互感。
- 拖拽时反馈明确。
- 不破坏输入输出面板的轻量感。

拖拽分隔条是工具站高级感的重要细节，它让用户可以根据文本长度自由调整输入和输出空间。

## 单栏查看按钮

界面支持“只看原文 / 只看结果”的单栏切换。这个设计让用户可以在需要时放大输入或输出区域，减少双栏带来的空间限制。

交互规则：

- 点击“只看原文”：隐藏输出区，只显示输入区。
- 点击“只看结果”：隐藏输入区，只显示输出区。
- 再次点击当前按钮：恢复分栏。
- 激活状态使用绿色强调，和成功反馈保持一致。

这个设计尤其适合长文本、大 JSON、大 YAML、大 Base64 内容。

## 状态反馈

状态反馈采用轻量提示条。

视觉规则：

- 成功状态使用浅绿色背景和绿色文字。
- 错误状态使用浅红背景和红色文字。
- 普通信息使用浅蓝背景和蓝色文字。
- 提示条位置靠近操作区。
- 反馈直接但不打断用户流程。

WebTools 不应频繁使用大弹窗。即时工具更适合使用轻量提示、Toast 或面板内状态条。

## 表格风格

如果页面需要展示结构化结果，可以使用轻量表格。

表格风格：

- 白色背景。
- 浅蓝灰表头。
- 细分隔线。
- 行高紧凑。
- Hover 行使用极浅蓝背景。
- 表头小字号加粗。
- 不使用厚重边框。
- 不使用复杂斑马纹。

表格只是辅助展示，不应该抢占输入输出编辑区的主导地位。

## 弹窗风格

WebTools 的弹窗应保持轻量。

弹窗适合：

- 设置。
- 错误详情。
- 下载确认。
- 大文本预览。

弹窗视觉：

- 白色背景。
- 中等圆角。
- 浅边框。
- 低透明遮罩。
- 按钮靠右。
- 内容简洁。

弹窗不应该承载主要编辑流程，主要编辑仍应发生在页面主体的输入输出面板中。

## 视觉层级

视觉层级主要通过以下方式建立：

- 顶部导航控制全局工具入口。
- 工具标题区说明当前上下文。
- 操作按钮形成主要行为区。
- 输入输出面板承载核心工作内容。
- 蓝色表示当前或主要操作。
- 绿色表示成功、激活或正向状态。
- 淡色边框和背景区分模块层级。

整体层级清楚但不夸张，始终让输入输出区域保持最高优先级。

## 响应式风格

桌面端：

- 顶部导航吸顶。
- 工具入口横向排列。
- 输入输出左右分栏。
- 分隔条可拖拽。
- 操作按钮靠近标题区。

窄屏端：

- 顶部导航横向滚动。
- 工具栏按钮自动换行。
- 输入输出改为上下布局。
- 单栏查看按钮继续可用。
- 编辑区域保持足够高度。

响应式目标是保证工具可用，而不是重新设计成复杂移动端应用。

## 应避免的设计

- 不要使用厚重后台 Sidebar。
- 不要做仪表盘。
- 不要使用复杂卡片堆叠。
- 不要让标题区占据太多高度。
- 不要把输入输出区域压缩得很小。
- 不要使用强阴影和高饱和渐变。
- 不要把按钮做成大号营销按钮。
- 不要用弹窗承载主要编辑流程。
- 不要使用大面积深色或沉重灰色背景。

## 最终风格总结

WebTools 的 UI 是一个浅色蓝绿系、顶部导航、全宽工作台、输入输出分栏、轻量面板、柔和圆角、等宽编辑器优先的开发者工具站界面。它的核心是快速粘贴、即时处理、低干扰查看结果，并通过蓝色和绿色建立清晰但克制的操作层级。
