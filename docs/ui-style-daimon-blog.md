# Daimon's Blog UI 风格说明

> 参考站点：https://blog.daimona.cn/  
> 定位：个人技术博客 / 轻量内容站 / Astro + Fuwari 风格博客界面。本文只描述 UI 风格、颜色、排版、导航、文章卡片、搜索、主题设置、页脚等视觉与交互特征，不描述文章内容本身。

## 整体风格

Daimon's Blog 整体是浅色、柔和、内容优先的个人技术博客风格。它不像传统后台管理系统，也不像工具站那样强调输入输出操作，而是更偏“阅读型内容站”和“个人主页型博客”。

整体视觉气质是：

- 轻量。
- 柔和。
- 清爽。
- 有个人风格。
- 内容阅读优先。
- 科技感适中。
- 不厚重、不商业化。

页面既有技术博客的结构感，也有个人主页的情绪表达。顶部导航、Hero 区、头像、标签、文章列表和底部播放器共同构成一种“个人数字空间”的感觉。

整体关键词可以概括为：

```text
浅色背景、柔和主题色、个人博客、内容卡片、顶部 Dock 导航、Hero 头像、文章流、圆角胶囊、轻量动效、主题切换、搜索面板、阅读优先、Fuwari 风格
```

## 色彩体系

Daimon's Blog 使用的是可调主题色体系。页面中可以看到主题色滑块，当前主题色数值为 105，说明颜色不是固定单一蓝绿，而是围绕一个可调 Hue 生成主题色。

整体颜色倾向：

- 页面背景是非常浅的低饱和色。
- 当前可见背景接近浅绿色 / 浅黄绿色调。
- 主体文字使用深色，保证阅读对比度。
- 链接、按钮、标签和高亮元素使用主题色。
- 卡片和导航使用半透明白色或浅色背景。
- 暗色模式可切换，但默认视觉是浅色阅读风格。

### 背景色

背景不是纯白，而是轻微带色相的浅色背景。它给页面提供柔和氛围，但不会抢占文章内容。

视觉特点：

- 明亮。
- 柔和。
- 低饱和。
- 不使用沉重灰色。
- 不使用大面积纯黑。
- 背景存在轻微动态或装饰层。

页面里可以看到 `animated-background`，说明背景不是完全静态白板，而是带有细微动态背景或视觉层次。

### 主题色 / Theme Color

主题色是这个博客的重要个性化特征。

它不是写死的单一主色，而是可以通过主题色滑块调整。主题色会影响：

- 当前高亮。
- 标签色。
- 按钮色。
- 链接色。
- 部分导航状态。
- 文章卡片交互反馈。

这种设计比普通博客更个性化，也更像 Fuwari 这类现代博客主题。

### 明暗模式

页面支持三种显示模式：

- 亮色。
- 暗色。
- 跟随系统。

这属于现代博客常见的 Display Settings 设计。主题切换入口位于顶部导航控制区，不影响阅读主体布局。

## 总体排版

Daimon's Blog 采用居中内容流布局，而不是全宽工作台布局。

这和 WebTools / Clash Template 不同：

- 工具站需要全宽输入输出区域。
- 博客站需要更合适的阅读宽度。

页面主体最大宽度约为 1024px，内容居中显示。这样可以保证文章标题、摘要、封面和标签在桌面端有良好阅读节奏，不会因为横向过宽导致阅读疲劳。

整体结构大致是：

```text
顶部 Dock 导航
Hero 个人介绍区
文章列表流
查看更多 / 分页
页脚版权信息
底部音乐播放器
返回顶部按钮
```

排版核心是纵向阅读流，而不是横向操作流。

## 顶部导航

顶部导航是 Dock 风格的悬浮导航栏。

它不是传统的全宽 Header，也不是后台 Sidebar，而是一个居中、圆润、半透明、带 Dock 感的导航条。

导航内容包括：

- Daimon's Blog 品牌入口。
- 主页。
- 归档。
- 关于。
- RSS。
- 探针。
- 常用工具。
- 搜索框。
- 主题 / 显示设置。

视觉特征：

- Sticky 吸顶。
- 居中显示。
- 半透明背景。
- 圆角明显。
- 项目之间留有适度间距。
- 当前项或 hover 项有轻量反馈。
- 图标和文字共同构成导航项。
- 移动端会隐藏部分导航并保留必要入口。

这种顶部导航比普通博客导航更现代，也更有个人站点的精致感。

## Dock 导航 / Dock Navbar

页面中导航元素使用了类似 `dock-navbar`、`dock-item`、`dock-link`、`dock-button` 这样的结构，整体更接近 Dock 样式。

Dock 风格特点：

- 导航项像一组圆角按钮。
- 每个入口有独立 hover / active 反馈。
- 整体不是一条硬边矩形导航栏。
- 导航条有浮动感和轻微玻璃感。
- 控件聚合在一个紧凑区域中。

适合个人博客，因为它既保留功能入口，又不会显得像企业后台。

## 品牌区与 Logo

品牌区位于顶部导航左侧。

可见品牌文字：

```text
Daimon's Blog
```

品牌区设计特点：

- 使用文字品牌为主。
- 左侧带有图标符号。
- 与导航项在同一条 Dock 导航中。
- 没有大面积 Logo 展示。
- 品牌识别低调但清楚。

这个博客更强调个人名称和内容气质，而不是复杂 Logo 图形。

## 搜索框

搜索是顶部导航里的重要功能。

可见特征：

- 搜索框位于导航右侧控制区。
- Placeholder 是“搜索”。
- 显示快捷键提示 `⌘ K`。
- 搜索框背景半透明。
- 聚焦时宽度会扩展。
- 移动端可能变为单独搜索按钮。

这种搜索设计属于现代文档站 / 博客站常见交互。

它传达的信息是：

- 搜索是高频入口。
- 支持键盘快捷键。
- 不打断页面阅读。
- 默认低存在感，聚焦时增强。

## 主题设置面板

页面顶部存在 Display Settings 控件。

可见内容包括：

- 亮色。
- 暗色。
- 跟随系统。
- 主题色。
- 重置默认。
- 主题色滑块。

这是一个浮动设置面板，不是普通表单页。

视觉特点：

- 面板从导航控制区展开。
- 使用浮层形式。
- 操作集中。
- 控件紧凑。
- 与整体圆角、半透明风格一致。

这种设计让博客具备个性化调色能力，增强个人站点属性。

## Hero 区

首页顶部有明显的 Hero 区。

Hero 区内容包括：

- 头像。
- 主标题文案。
- GitHub 链接。
- 个人标签。

当前主标题是：

```text
有些人像黄昏，来时温柔，去时荒凉
```

Hero 区不是营销页那种巨大 Banner，而是个人博客的身份展示区。

视觉特点：

- 头像作为视觉中心之一。
- 标题具有个人表达和情绪感。
- 标签用于展示个人身份和兴趣。
- 整体居中或接近居中。
- 视觉比文章列表更有个人主页感。

## 头像 / Avatar

头像是 Hero 区的重要视觉元素。

特征：

- 图片形式。
- 点击可进入关于页面。
- 与个人介绍绑定。
- 作为页面第一视觉锚点。

头像让博客从“技术文章列表”变成“个人内容空间”。

## 个人标签 / Personal Tags

Hero 区下方有一组个人标签，例如：

- VPS 驯兽师。
- 逆向爱好者。
- 服务器炼金术士。
- NixOS 信徒。
- Root 玩家。
- 终端效率洁癖。

这些标签使用导航型链接，视觉上接近圆角标签或轻量文本胶囊。

作用：

- 建立作者人设。
- 快速传达博客主题。
- 增加个人风格。
- 作为归档入口。

## 文章列表布局

文章列表是首页主体。

每篇文章以文章卡片 / 文章链接块形式呈现。

结构包括：

- 封面图。
- 文章标题。
- 摘要。
- 日期。
- 字数。
- 标签。

当前文章列表是单列纵向流，适合阅读型博客。

视觉特点：

- 内容宽度固定在合适阅读范围。
- 每篇文章之间有明显间距。
- 封面图占据较大视觉区域。
- 标题层级清楚。
- 摘要不喧宾夺主。
- 日期、字数、标签属于次级信息。

## 文章卡片 / Post Card

文章卡片是博客首页最重要的内容组件。

可见结构：

```text
封面图
文章标题
文章摘要
日期 / 字数 / 标签
```

文章卡片整体更像“可点击的内容块”，而不是传统厚重卡片。它不依赖明显边框和强阴影，而是通过封面图、间距、文字层级和 hover 效果建立可点击感。

视觉特点：

- 封面图大。
- 标题明显。
- 摘要简洁。
- 元信息弱化。
- 标签使用 `#tag` 文本形式。
- 不使用复杂按钮。

## 封面图 / Cover Image

封面图在文章卡片中占据较大面积。

作用：

- 建立文章视觉识别。
- 增强列表可浏览性。
- 打破纯文本列表单调感。
- 让首页更像现代博客，而不是传统文字归档。

封面图风格：

- 宽图。
- 横向比例明显。
- 适合技术主题封面。
- 图像本身承担文章气质表达。

## 文章标题 / Post Title

文章标题使用二级标题层级。

视觉特点：

- 字重较高。
- 比摘要和元信息更醒目。
- 不使用夸张大字号。
- 保持博客阅读节奏。

标题是文章卡片的主要点击动机。

## 文章摘要 / Excerpt

摘要用于解释文章内容。

视觉特点：

- 颜色比标题弱。
- 字号适中。
- 行数不宜过多。
- 主要帮助用户判断是否点击。

摘要不应该抢占标题和封面图的视觉权重。

## 文章元信息 / Meta Info

文章元信息包括：

- 发布日期。
- 字数。
- 标签。

视觉特点：

- 小字号。
- 次级颜色。
- 使用斜杠或间距分隔。
- 标签以 `#` 开头。

这类信息是辅助阅读决策的，不是视觉主角。

## 标签 / Tags

博客标签使用轻量文本标签形式。

例如：

```text
#VPS #Linux #SSH #BBR #安全
```

这种标签比彩色 Pill 更克制，更适合文章列表。它不会让页面变得像后台系统，也不会破坏阅读流。

## 分页 / Pagination

首页底部有分页区域。

可见形式：

- 当前页 `1`。
- 后续页 `2`、`3`、`4`。
- 下一页入口。

分页视觉比较轻量，不喧宾夺主。它服务于文章浏览，不承担强交互视觉。

## 查看完整目录按钮

文章列表后有“查看完整目录”入口。

视觉特点：

- 圆角胶囊按钮。
- 背景使用浅主题色。
- 居中或靠近内容流。
- 作为归档页入口。

这是博客内容流和归档结构之间的过渡按钮。

## 页脚 / Footer

页脚包含：

- 版权协议。
- 博客框架信息。
- Fuwari 主题信息。
- ICP 备案。

视觉特点：

- 文本较小。
- 颜色弱化。
- 链接保持可点击样式。
- 位于内容流末尾。
- 不使用厚重页脚背景。

页脚保持信息完整，但不抢占页面视觉。

## 返回顶部按钮 / Back to Top

页面有返回顶部按钮。

作用：

- 长列表阅读后快速回到顶部。
- 提升博客浏览体验。

视觉应保持轻量，通常是悬浮按钮或小图标按钮。

## 音乐播放器 / Audio Player

页面底部存在音乐播放器。

它是博客个性化的重要组成部分。

可见元素包括：

- 歌曲名。
- 艺术家。
- 播放进度。
- 控制按钮。
- 播放时间。

视觉定位：

- 它不是主要内容。
- 它增加个人空间感。
- 应该固定或浮动在底部。
- 控件要紧凑，不遮挡阅读主体。

这个设计让博客更像个人主页，而不是纯技术文档站。

## 动效风格

页面中有 `animated-background`、导航 hover、搜索框聚焦扩展、按钮 active scale 等轻量动效。

动效特点：

- 轻微。
- 柔和。
- 不炫技。
- 用于增强交互反馈。
- 不影响阅读。

适合的动效包括：

- Hover 背景变化。
- 按钮按下轻微缩放。
- 搜索框展开。
- 浮层淡入。
- 返回顶部平滑滚动。

## 暗色模式

博客支持暗色模式。

暗色模式不是单独风格，而是同一套主题的另一种显示状态。

暗色模式应该保持：

- 文字可读。
- 背景不纯黑。
- 面板有层次。
- 主题色仍然可见。
- 图片和文章内容不被过度压暗。

## 响应式风格

博客需要适配桌面和移动端。

桌面端：

- 顶部 Dock 导航居中。
- 搜索框完整显示。
- 主体宽度约 1024px。
- 文章封面图较大。
- 文章列表单列阅读流。

移动端：

- 部分导航项隐藏或折叠。
- 搜索可能变成按钮。
- 内容宽度贴近屏幕。
- 文章卡片保持单列。
- 顶部导航仍保持轻量。

响应式目标是保证阅读体验，而不是强行塞下所有桌面导航项。

## 与工具站风格的区别

Daimon's Blog 和 WebTools / Clash Template 的区别很明显。

WebTools / Clash Template：

- 强调输入输出。
- 强调操作效率。
- 强调全宽工作台。
- 适合横向双栏。

Daimon's Blog：

- 强调阅读。
- 强调个人表达。
- 强调文章列表。
- 适合居中内容流。
- 顶部导航更像 Dock。
- Hero 区更重要。

所以这个博客不应该改成全宽工作台，也不应该使用后台 Sidebar。

## 应避免的设计

- 不要使用后台管理 Sidebar。
- 不要使用全宽表格工作台。
- 不要把文章列表做得像数据表。
- 不要使用厚重卡片和强阴影。
- 不要使用高饱和霓虹色。
- 不要让顶部导航太高。
- 不要让搜索框和主题设置抢占主要视觉。
- 不要移除 Hero 区的个人表达。
- 不要让音乐播放器遮挡阅读主体。
- 不要把页脚做成厚重商业网站页脚。

## 最终风格总结

Daimon's Blog 的 UI 是一个浅色低饱和、主题色可调、顶部 Dock 导航、Hero 个人介绍、单列文章流、大封面图文章卡片、轻量搜索与主题设置、底部音乐播放器组成的现代个人技术博客界面。

它的核心不是“管理”和“操作”，而是“阅读”和“个人表达”。整体应保持柔和、清爽、低干扰、有技术气质，也有个人生活感。
