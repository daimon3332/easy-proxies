# 前端 UI 术语速查教程

> 目标：帮助非前端背景的人快速理解在描述页面时常用的 UI / UX / 前端术语。本文重点覆盖这次沟通中反复涉及的颜色、导航、布局、卡片、表格、弹窗、编辑器、工作台等概念。

## 1. 色彩相关术语

### 色彩体系 / Color System

色彩体系指一个网站或应用统一使用的一组颜色规则，包括主色、辅助色、背景色、文字色、边框色、成功色、警告色、错误色等。

常见描述：

- 这个 UI 是浅蓝绿色系。
- 主色是青绿色，辅助色是天空蓝。
- 错误状态用浅红色。
- 背景是浅蓝灰，不是纯白。

### 主色 / Primary Color

主色是页面最核心的品牌颜色，通常用于：

- 主按钮
- 当前导航项
- 聚焦状态
- 重要高亮

例如：蓝色、青绿色都可以作为主色。

### 辅助色 / Secondary Color

辅助色用于补充主色，降低单一颜色带来的单调感。

例如：

- 主色是青绿色。
- 辅助色是浅蓝色。

### 强调色 / Accent Color

强调色用于表达特殊状态，例如成功、激活、提醒。

例如：

- 绿色表示成功。
- 黄色表示等待。
- 红色表示失败。

### 青绿 / Teal

青绿是介于蓝色和绿色之间的颜色。它经常用于现代 SaaS、工具站、科技感界面。

适合表达：

- 干净
- 现代
- 成功
- 稳定
- 轻科技感

### 浅蓝 / Light Blue

浅蓝常用于工具站、后台面板、开发者工具。

适合表达：

- 清爽
- 安全
- 轻量
- 信息科技感

### 低饱和颜色 / Low Saturation Color

低饱和颜色是不刺眼、不浓烈的颜色。

例如：

- 浅蓝灰
- 淡青色
- 浅绿色
- 浅红色

低饱和颜色适合长期使用的工具界面，因为不容易疲劳。

### 浅色基底 / Light Base

浅色基底指页面整体以浅色为基础，例如白色、浅蓝、浅灰、浅青。

常见描述：

- 页面是浅色基底。
- 整体明亮干净。
- 不使用深色大面积铺底。

### 渐变 / Gradient

渐变是颜色从一种平滑过渡到另一种。

常见类型：

- 线性渐变 / Linear Gradient
- 径向渐变 / Radial Gradient

常见描述：

- 蓝绿渐变按钮。
- 背景有浅蓝到浅青的渐变。
- Logo 使用青绿到天空蓝渐变。

### 径向光晕 / Radial Glow

径向光晕是背景中类似柔和光斑的效果。

常见描述：

- 左上角有淡青色光晕。
- 右上角有淡蓝色光晕。
- 页面背景有轻微科技感。

### 文字色 / Text Color

文字色分为：

- 主文字色：标题和主要内容。
- 次级文字色：说明、提示、辅助信息。
- 弱文字色：不重要的说明或占位信息。

常见描述：

- 主文字使用深蓝灰。
- 次级文字使用灰蓝色。
- 辅助说明降低存在感。

### 边框色 / Border Color

边框色用于分隔区域。工具站通常使用浅蓝灰边框，不使用厚重黑边。

常见描述：

- 淡色边框。
- 浅蓝灰分隔线。
- 边框只负责区分区域，不制造强烈割裂。

## 2. 导航相关术语

### 导航栏 / Navigation Bar / Navbar

导航栏是页面中用于切换功能或页面的区域。

它可以在顶部，也可以在左侧。

常见描述：

- 顶部导航栏。
- 左侧导航栏。
- 当前页面高亮。

### 顶部导航 / Top Navigation

顶部导航位于页面顶部，横向排列入口。

适合：

- 工具站
- 文档站
- 在线转换工具
- 页面数量不太多的应用

常见描述：

- 顶部导航采用扁平工具栏设计。
- 顶部导航不占用过多垂直空间。
- 当前页面使用蓝色高亮。

### 侧边栏 / Sidebar

侧边栏是位于页面左侧或右侧的导航区域，后台系统中最常见。

适合：

- 管理后台
- 控制台
- 节点管理
- 多模块系统

常见描述：

- 左侧 Sidebar 固定。
- Sidebar 中显示导航和数量 Badge。
- 当前菜单项高亮。

### 导航项 / Nav Item

导航项是导航栏中的一个按钮或链接。

例如：

- 导入节点
- 节点池
- 设置
- JSON 格式化

### 当前项 / Active Item

当前项表示用户现在所在的页面。

常见表现：

- 蓝色背景。
- 蓝绿渐变背景。
- 字体加粗。
- 边框高亮。

### Hover 状态

Hover 是鼠标移到元素上时的状态。

常见表现：

- 背景变浅蓝。
- 文字变蓝。
- 边框变亮。
- 按钮轻微上浮。

### 品牌区 / Brand Area

品牌区通常位于导航左侧或侧边栏顶部，包含 Logo 和网站名称。

常见结构：

```text
[Logo] WebTools
[Logo] Easy Proxies
[Logo] Clash Template
```

### Logo

Logo 是网站或应用的标识。

常见描述：

- 圆角方形 Logo。
- 蓝绿渐变 Logo。
- 白色字母 C / W。
- 与顶部导航风格一致。

### Favicon

Favicon 是浏览器标签页上的小图标。

常见描述：

- Favicon 与 Logo 保持一致。
- 使用 SVG 格式。
- 浏览器标签页显示圆角蓝绿图标。

### Badge / 徽标

Badge 是小型数量或状态标记，常出现在导航项或按钮旁边。

例如：

```text
节点池 169
失败节点 339
```

常见风格：

- 胶囊形。
- 浅蓝背景。
- 小字号。

## 3. 页面排版相关术语

### 页面排版 / Page Layout

页面排版指页面整体如何组织，包括导航、标题、内容区、工具栏、表格、弹窗等的位置关系。

常见描述：

- 页面采用全宽工作台排版。
- 页面不是居中窄容器。
- 左侧导航，右侧内容。

### 全宽布局 / Full-width Layout

全宽布局指内容区域尽量使用浏览器宽度，而不是被限制在中间一小块。

适合：

- 表格很多的页面
- 编辑器页面
- 输入输出双栏页面
- 工作台页面

常见描述：

- 页面充分利用横向空间。
- 不要把内容挤在中间。
- 宽屏下左右双栏展示。

### 全宽工作台 / Full-width Workspace

全宽工作台是比全宽布局更具体的说法，强调页面是给用户“操作”的，不是给用户“看展示”的。

常见结构：

```text
左侧输入区                  右侧输出区
配置表单                    预览结果
操作按钮                    编辑器
```

适合：

- YAML 配置生成
- JSON 格式化
- URI 转换
- 代理节点管理

### 工作台布局 / Workspace Layout

工作台布局强调工具操作流程。

特点：

- 输入、操作、输出在同一屏完成。
- 按钮靠近操作区域。
- 编辑区或表格占据主要空间。
- 页面不做过多装饰。

### 双栏布局 / Two-column Layout

双栏布局是页面分成左右两列。

常见形式：

```text
左侧输入       右侧输出
左侧配置       右侧预览
左侧列表       右侧详情
```

适合：

- 输入输出对照
- 配置生成
- 编辑预览

### 左右双栏 / Side-by-side Panels

左右双栏强调输入和输出并排展示。

常见描述：

- 左侧粘贴原文。
- 右侧显示结果。
- 用户从左到右完成处理。

### 上下分栏 / Vertical Split Layout

上下分栏是输入在上、输出在下。

适合：

- 短文本
- 命令行转换
- URL 编解码
- 不适合被左右压缩的内容

### 居中窄容器 / Centered Narrow Container

居中窄容器是传统展示页或表单页常见布局。

常见问题：

- 宽屏浪费空间。
- 输入框太窄。
- 表格显示不全。
- 大文本编辑体验差。

在工具站、工作台、表格管理页中通常不推荐。

### 响应式布局 / Responsive Layout

响应式布局指页面在不同屏幕宽度下自动调整。

常见变化：

- 桌面端左右双栏。
- 移动端上下堆叠。
- 顶部导航横向滚动。
- 表格变成卡片式列表。

## 4. 容器与面板术语

### 卡片 / Card

卡片是承载一组内容的容器。

常见特征：

- 白色或半透明白色背景。
- 圆角。
- 细边框。
- 柔和阴影。
- 内部有标题和内容。

适合：

- 表单分组
- 设置项
- 来源列表
- 状态块

### 面板 / Panel

面板和卡片类似，但更偏工具区域。

例如：

- 输入面板
- 输出面板
- 日志面板
- 编辑器面板

面板通常比卡片更强调功能区域，而不是展示内容。

### 工具面板 / Tool Panel

工具面板是在线工具站中用于输入、输出、操作的区域。

常见结构：

```text
面板标题
输入区域 / 输出区域
操作按钮
```

### 卡片头部 / Card Header

卡片头部是卡片顶部区域，通常包含标题、说明、按钮。

常见描述：

- 卡片头部左侧显示标题。
- 右侧放操作按钮。
- 底部有浅色分割线。

### 卡片内容区 / Card Body

卡片内容区是卡片主体。

常见内容：

- 表单
- 表格
- 输入框
- 文本说明
- 列表

### 分隔线 / Divider

分隔线用于区分模块。

常见描述：

- 浅蓝灰分隔线。
- 1px 细线。
- 不制造强烈割裂。

### 柔和阴影 / Soft Shadow

柔和阴影用于让卡片浮起来一点，但不厚重。

常见描述：

- 阴影轻柔。
- 只负责分层。
- 不制造压迫感。

### 玻璃感 / Glassmorphism

玻璃感是一种半透明、带背景模糊的视觉效果。

常见特征：

- 半透明白色背景。
- backdrop blur。
- 淡边框。
- 背景光晕能透出来。

注意：轻量工具站可以有轻微玻璃感，但不应该过度。

### 白底 / White Surface

白底是工具面板最常见的背景。

适合：

- 输入区
- 输出区
- 表格
- 弹窗

### 浅色面板 / Light Panel

浅色面板通常是白色、浅蓝白、浅灰白。

它能让文本和表格更容易阅读。

## 5. 圆角与形状术语

### 圆角 / Border Radius

圆角是元素边角的弧度。

常见元素：

- 按钮
- 输入框
- 卡片
- 弹窗
- Logo
- 标签

### 大圆角 / Large Radius

大圆角让界面更柔和。

适合：

- 浅蓝绿色工具站
- SaaS 风格页面
- 轻量卡片

### 胶囊形 / Pill Shape

胶囊形是左右完全圆的长条形。

常见于：

- 标签
- Badge
- 状态标记
- 小按钮

例如：

```text
成功
失败
节点池 169
```

### 图标按钮 / Icon Button

图标按钮是只显示图标或图标加文字的按钮。

常见用途：

- 删除
- 复制
- 下载
- 刷新

---

## 6. 按钮相关术语

### 按钮层级 / Button Hierarchy

按钮层级指不同按钮的重要程度。

常见层级：

- 主按钮
- 次主按钮
- 普通按钮
- 危险按钮
- 禁用按钮

### 主按钮 / Primary Button

主按钮用于最重要操作。

常见样式：

- 蓝色实底。
- 蓝绿渐变。
- 白色文字。

### 次主按钮 / Secondary Button

次主按钮用于重要但不是最核心的操作。

常见样式：

- 蓝色边框。
- 浅蓝背景。
- 蓝色文字。

### 普通按钮 / Default Button

普通按钮用于常规操作。

常见样式：

- 白底。
- 浅边框。
- 灰蓝文字。

### 危险按钮 / Danger Button

危险按钮用于删除、清空等操作。

常见样式：

- 红色文字。
- 浅红背景。
- 浅红边框。

不建议默认使用大面积纯红，除非是最终确认。

### 禁用状态 / Disabled State

禁用状态表示按钮暂时不能点击。

常见表现：

- 透明度降低。
- 鼠标不可点击。
- 文字变浅。

### 激活态 / Active State

激活态表示当前按钮代表正在使用的状态。

例如：

- 当前页面导航项。
- 只看结果按钮被激活。
- 当前筛选项。

## 7. 表单与输入相关术语

### 表单 / Form

表单是一组输入控件。

包括：

- 输入框
- 下拉框
- 复选框
- 单选框
- 文本域
- 提交按钮

### 输入框 / Input

输入框用于短文本。

例如：

- 标签名
- 端口号
- URL
- 用户名

### 文本域 / Textarea

文本域用于多行文本。

适合：

- URI 列表
- YAML
- JSON
- Base64
- 日志

### 下拉框 / Select

下拉框用于从固定选项中选择。

例如：

- 国家
- 标签
- 状态
- 导入格式

### 复选框 / Checkbox

复选框用于开关或多选。

例如：

- 是否自动加入节点池。
- 表格行多选。
- 是否启用自动刷新。

### 输入标签 / Label

Label 是输入框上方或左侧的字段名称。

例如：

```text
Tag 前缀
Multi 起始端口
刷新间隔
```

### 占位符 / Placeholder

Placeholder 是输入框为空时显示的提示文字。

例如：

```text
在此粘贴 JSON...
每行一个订阅链接...
```

### 聚焦状态 / Focus State

聚焦状态表示输入框正在被编辑。

常见表现：

- 边框变蓝。
- 出现浅蓝外发光。
- 光标进入输入框。

---

## 8. 编辑器相关术语

### 代码编辑器式视觉 / Code Editor Style

代码编辑器式视觉指输入区域看起来像编辑器，而不是普通表单。

特征：

- 等宽字体。
- 行高舒适。
- 大面积文本区域。
- 适合长配置文本。

### 等宽字体 / Monospace Font

等宽字体是每个字符宽度相同的字体。

适合：

- YAML
- JSON
- URI
- Base64
- 命令行
- 日志

常见字体：

- Cascadia Mono
- Consolas
- SFMono-Regular

### CodeMirror

CodeMirror 是常见的前端代码编辑器组件。

常见能力：

- 语法高亮
- 行号
- 括号匹配
- 代码折叠
- 编辑器主题

### 行号区域 / Gutter

Gutter 是代码编辑器左侧显示行号的区域。

常见描述：

- 行号区域使用浅灰背景。
- Gutter 和正文之间有细分隔线。

### 语法高亮 / Syntax Highlighting

语法高亮是根据内容类型给不同语法上色。

例如：

- JSON key 一种颜色。
- 字符串一种颜色。
- 数字一种颜色。

---

## 9. 表格相关术语

### 表格 / Table / Data Table

表格用于展示结构化数据。

适合：

- 节点列表
- 端口列表
- 任务列表
- 来源列表

### 表头 / Table Header

表头是表格顶部列名。

例如：

```text
名称 / 国家 / 延迟 / 状态 / 端口 / 操作
```

### 表格行 / Table Row

表格行是一条数据。

例如一个节点、一条订阅来源、一个端口。

### 表格列 / Table Column

表格列是某一类字段。

例如：名称列、国家列、延迟列。

### 全选框 / Select All Checkbox

全选框通常在表头第一列，用于选择全部行。

### 行选择框 / Row Checkbox

每行前面的复选框，用于选择单条数据。

### 批量操作 / Bulk Action

批量操作是对多选数据一起执行操作。

例如：

- 批量测速
- 批量删除
- 批量加入节点池

### 排序 / Sorting

排序是改变表格显示顺序。

常见交互：

- 点击列头升序。
- 再点击一次降序。
- 当前排序列高亮。

### 筛选 / Filtering

筛选是减少显示范围。

例如：

- 只看某个国家。
- 只看某个标签。
- 只看失败节点。

### 搜索框 / Search Box

搜索框用于按关键词过滤列表。

例如：

- 搜索节点名称。
- 搜索标签。

### 空状态 / Empty State

空状态是没有数据时显示的提示。

例如：

```text
还没有导入节点
暂无失败节点
```

## 10. 弹窗与反馈术语

### 弹窗 / Modal / Dialog

弹窗是覆盖在页面上的交互层。

常见用途：

- 确认操作
- 编辑内容
- 显示结果
- 显示进度

### 模态框 / Modal

Modal 通常会阻止用户操作后面的页面。

适合：

- 重要确认
- 登录
- 任务进度
- 大文本编辑

### 对话框 / Dialog

Dialog 是弹窗主体内容区域。

常见内容：

- 标题
- 正文
- 操作按钮

### 阻塞式弹窗 / Blocking Dialog

阻塞式弹窗表示任务没完成前不关闭。

适合：

- 批量测速
- 导入节点
- 刷新订阅
- 长任务进度

### 进度弹窗 / Progress Dialog

进度弹窗用于显示任务进度。

常见内容：

```text
进度 49/1800
成功 35
失败 14
入池 35
```

### Toast

Toast 是页面角落短暂出现的小提示。

适合：

- 保存成功
- 复制成功
- 小错误提示

不适合：

- 长任务
- 批量操作进度

### 状态提示条 / Status Bar / Alert

状态提示条通常位于操作区附近。

常见状态：

- 成功
- 错误
- 警告
- 普通信息

### 遮罩 / Overlay / Backdrop

遮罩是弹窗后面的半透明背景。

作用：

- 弱化后方页面。
- 强调当前弹窗。
- 阻止误操作。

## 11. 工具站特殊术语

### 工具站 / Tool Site

工具站是提供在线工具的网站。

例如：

- JSON 格式化
- YAML 转换
- Base64 编解码
- Clash 配置生成

### 工具栏 / Toolbar

工具栏是一组操作按钮。

例如：

```text
格式化 / 压缩 / 复制 / 下载 / 清空
```

### 输入输出分栏 / Input-output Split

输入输出分栏是工具站常见布局。

左侧输入，右侧输出，方便对照。

### 拖拽分隔条 / Resizable Splitter

拖拽分隔条用于调整输入区和输出区宽度。

常见描述：

- 分隔条很细。
- Hover 时变浅蓝。
- 拖动时改变左右比例。

### 只看原文 / Source Only

只显示输入区，隐藏输出区。

适合编辑大文本。

### 只看结果 / Result Only

只显示输出区，隐藏输入区。

适合检查和复制结果。

### 预览区 / Preview Panel

预览区用于显示处理结果。

例如：

- YAML 输出预览
- HTML 渲染预览
- 图片预览

### 输出区 / Output Panel

输出区显示工具处理后的结果。

### 输入区 / Input Panel

输入区用于粘贴或编辑原始内容。

## 12. 常见风格描述模板

### 描述颜色

可以这样说：

```text
整体使用浅色蓝绿系，主色是青绿色和天空蓝，背景为浅蓝灰或浅蓝绿渐变，整体低饱和、清爽、低干扰。
```

### 描述布局

可以这样说：

```text
页面采用全宽工作台布局，不使用居中窄容器。桌面端使用左右双栏，左侧输入，右侧输出，充分利用横向空间。
```

### 描述导航

可以这样说：

```text
顶部导航采用扁平工具栏设计，左侧是 Logo 和品牌名，右侧是横向导航项，当前页面使用蓝色高亮。
```

或：

```text
页面采用左侧 Sidebar 导航，导航项显示数量 Badge，当前项使用浅蓝绿渐变高亮。
```

### 描述卡片

可以这样说：

```text
卡片使用半透明白底、淡蓝灰边框、大圆角和柔和阴影，整体有轻微玻璃感，但不厚重。
```

### 描述表格

可以这样说：

```text
节点列表使用 Data Table，表头支持点击排序，每行前面有复选框，表头有全选框，批量操作按钮放在表格上方。
```

### 描述弹窗

可以这样说：

```text
耗时操作使用阻塞式进度弹窗，显示实时进度、成功数量、失败数量和最终结果，完成前不自动关闭。
```
