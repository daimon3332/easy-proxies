# Easy Proxies UI 风格说明

## 整体风格

Easy Proxies 是浅蓝绿色的轻量代理节点控制台风格。它比 WebTools 更偏管理面板，比传统后台更轻、更柔和。整体目标是让用户清楚地管理大量节点、查看状态、执行批量操作，并且在耗时任务中获得明确反馈。

它不应该是沉重的企业后台，也不应该是花哨的展示型网页。它应该像一个长期运行的工具台：信息清楚、操作直接、状态明确、视觉低干扰。

整体关键词：浅蓝绿、轻后台、工具控制台、左侧导航、全宽内容、卡片分区、表格管理、圆角柔和、弹窗实时进度、端口清晰、日志全屏。

## 色彩体系

Easy Proxies 使用浅蓝绿色系。

主色关系：

- 浅蓝色用于导航高亮、聚焦、辅助主操作。
- 绿色用于成功、可用、入池、正向操作。
- 黄色用于等待、测速中、处理中。
- 红色用于失败、删除、危险操作。
- 灰蓝色用于说明文字、弱提示、次级信息。

页面背景不是纯白，而是浅蓝绿渐变，并带有非常淡的径向光晕。这样可以减少纯后台系统的冰冷感，同时保持工具界面的干净和可读性。

整体色彩比 Clash Template 更偏“管理工具”，比 WebTools 更柔和、更有卡片层次。

## 总体排版

Easy Proxies 当前采用左侧导航加右侧内容区的后台工具布局。

页面结构是：

- 左侧 Sidebar：功能导航和状态数量。
- 右侧 Main Content：当前页面标题、说明、表格、卡片、弹窗入口等。

右侧内容区应占满剩余宽度，不应该居中成窄容器。节点表格、日志、端口状态都需要横向空间，因此页面必须是全宽内容布局。

主体排版原则：

- 不做仪表盘。
- 不做三列小卡片堆叠。
- 不把主要内容挤在页面中央。
- 表格、日志、端口列表尽量使用整块宽区域。
- 批量操作按钮靠近对应表格。

## 左侧导航 Sidebar

左侧 Sidebar 是 Easy Proxies 的核心导航结构。

视觉特征：

- 固定在左侧。
- 宽度适中。
- 半透明白底。
- 带轻微背景模糊。
- 右侧使用浅色边框分隔。
- 顶部是品牌区。
- 下方是垂直导航列表。

导航项应显示关键数量，例如候选节点、节点池、失败节点数量。这样用户不用进入页面，也能看到系统核心状态。

导航项状态：

- 默认：透明背景、灰蓝文字。
- Hover：浅蓝背景、蓝色文字。
- Active：浅蓝绿渐变背景、深色文字、轻描边。
- 数量 Badge：胶囊形浅蓝底。

## Logo 与品牌区

Logo 位于 Sidebar 顶部。

风格要求：

- 圆角方形。
- 蓝绿视觉或项目图标。
- 尺寸适中。
- 与浅色背景协调。
- 旁边显示项目名称和简短副标题。

品牌区只负责识别，不应占用太多垂直空间。

## 页面标题区

右侧顶部是页面标题区。

视觉要求：

- 标题清楚但不夸张。
- 一行简短说明解释当前页面用途。
- 不使用大型 Hero 区。
- 不使用仪表盘欢迎语。
- 页面标题下方直接进入主要操作区域。

标题区的作用是告诉用户当前在哪个模块，而不是承担装饰。

## 卡片风格

Easy Proxies 使用轻量卡片来组织页面内容。

卡片视觉：

- 白色或半透明白色背景。
- 淡蓝灰边框。
- 大圆角。
- 柔和阴影。
- 卡片头部有标题和右侧操作。
- 卡片内容区有明确内边距。

卡片比 WebTools 更明显，因为 Easy Proxies 有更多管理类信息；但卡片不能滥用。强相关内容应该放在一个卡片中，不要为了视觉分区拆成很多碎片。

## 表格风格

表格是 Easy Proxies 最重要的 UI 组件。

节点池、候选节点、失败节点都应该以表格方式呈现，而不是普通卡片列表。

表格视觉：

- 白底。
- 浅蓝边框。
- 圆角外框。
- 表头加粗。
- 行之间用浅色分割线。
- 横向可滚动。
- 单元格必须对齐。
- URI 等长文本不应默认完整展示，避免破坏表格可读性。

推荐列类型：

- 选择框。
- 序号。
- 节点名称。
- 国家。
- 延迟。
- 状态。
- 端口。
- 标签。
- 操作。

表格交互应该符合常规数据表：

- 表头有全选框。
- 每行有选择框。
- 点击列头排序。
- 再点一次反向排序。
- 当前排序列有蓝色高亮。
- 批量操作按钮放在表格上方。

## 筛选、排序与端口重排

Easy Proxies 必须明确区分筛选、排序和端口重排。

筛选是减少当前表格显示范围。它应该放在表格上方，使用小尺寸下拉框或搜索框，例如国家、标签、状态等。筛选框不应过长，也不应出现无意义字段。

排序是改变当前表格显示顺序。排序应该通过点击表头完成，不需要额外堆一排排序按钮。

端口重排是改变实际端口绑定，属于高影响操作。它只适合节点池页面，并且应该放在节点池表格上方。操作完成后必须有弹窗结果提示。

## 标签、Badge 与状态 Pill

节点状态、国家、延迟、端口等适合使用胶囊标签。

视觉规则：

- 成功、可用、入池使用绿色。
- 延迟正常可以使用绿色或蓝色。
- 等待、测速中使用黄色。
- 失败使用红色。
- 未知或未测试使用灰色。

这些颜色不能太浅，否则用户看不出状态差异。文字颜色应比背景更深，保证可读性。

## 按钮风格

按钮整体圆润、轻量、有明确层级。

按钮类型：

- 默认按钮：白底、浅蓝边框、深蓝灰文字。
- 主按钮：蓝色渐变、白字。
- 成功按钮：绿色渐变、白字。
- 危险按钮：白底或浅红底、红字。
- 小按钮：用于表格行操作，尺寸更紧凑。

按钮点击后必须有反馈。耗时操作不能只依赖按钮状态或 Toast，必须显示进度弹窗。

## 弹窗与进度反馈

Easy Proxies 的弹窗非常重要，因为节点测速、订阅刷新、批量测试都可能耗时很长。

弹窗视觉：

- 居中显示。
- 白色卡片主体。
- 大圆角。
- 淡蓝灰边框。
- 柔和阴影。
- 蓝灰半透明遮罩。

耗时任务应使用阻塞式进度弹窗：

- 任务开始时立即弹出。
- 显示当前操作名称。
- 显示实时进度。
- 显示成功、失败、入池等结果数量。
- 任务完成前不自动关闭。
- 完成后出现确定按钮。
- 失败时显示错误信息。

这是 Easy Proxies 区别于普通工具站的重要交互：它要让用户知道长时间任务是否仍在运行。

## Toast 提示

Toast 只适合轻量反馈，例如保存成功、小错误、非耗时提醒。

Toast 视觉：

- 右上角出现。
- 白底卡片。
- 浅边框。
- 左侧彩色竖线表示状态。
- 成功绿色，错误红色。

Toast 不应该用于替代批量测速、刷新订阅、导入节点这类长任务反馈。

## 导入页面视觉

导入页面应是工具输入页，而不是复杂后台表单。

视觉结构：

- 导入格式选择放在显眼位置。
- 标签前缀输入靠近导入内容。
- 自动加入节点池使用明确复选框。
- 大文本输入区足够宽。
- 导入并测试按钮靠近输入区。

不需要预览区，不需要示例填充按钮，不需要把格式选择放在角落。用户进入页面后应该马上知道先选格式、再粘贴、再导入。

## 设置页面视觉

设置页是配置和来源管理页面。

视觉要求：

- 输入框宽度适中，不要过长。
- 天、小时、分钟使用小输入框。
- 当前导入来源用列表行展示。
- 每个来源显示类型、标签、数量、URL、更新时间和操作按钮。
- 手动刷新按钮要明显。
- 删除按钮使用危险弱强调。

设置页不是主要数据表，但也不能变成碎片卡片堆叠。

## 端口状态页视觉

端口状态页的核心是确认实际监听端口和节点对应关系。

视觉重点：

- 顶部用文字说明扫描结果。
- 明确说明从哪个起始端口开始寻找。
- 明确说明节点数量、已找到端口数量、最后使用到哪个端口。
- 不可用端口单独汇总。
- 下方列表只显示端口和节点名称。

不要显示无意义字段，例如“节点占用”。不要使用看起来可以编辑但实际不能修改的假输入框。

## 日志页视觉

日志页应该接近全屏。

视觉要求：

- 日志卡片高度接近浏览器可视高度。
- 日志内容区占满卡片主体。
- 使用等宽字体。
- 背景为极浅蓝白。
- 支持清空日志。
- 不要让日志只占页面一小块，下面留大片空白。

日志页是排查问题的重要页面，不应该被卡片高度限制。

## 响应式风格

桌面端：

- Sidebar 固定在左侧。
- 右侧内容区全宽。
- 表格横向滚动。
- 日志和大文本输入尽量占满高度。

移动端：

- Sidebar 变为顶部区域。
- 导航按钮变成多列或换行。
- 表格行可以转为单列布局。
- 筛选器纵向排列。
- 按钮组允许换行。

响应式目标是保证可操作，而不是强行保持桌面表格形态。

## 应避免的设计

- 不要做仪表盘。
- 不要三列小卡片堆叠。
- 不要让表格列不对齐。
- 不要默认展示完整 URI。
- 不要让按钮点击后没有反馈。
- 不要用 Toast 代替耗时进度弹窗。
- 不要混淆排序、筛选和端口重排。
- 不要使用过长且无意义的输入框。
- 不要放无法编辑但看起来能编辑的控件。
- 不要让日志页只占页面一小块。

## 最终风格总结

Easy Proxies 的 UI 是一个浅蓝绿色、左侧导航、全宽内容、卡片分区、表格管理、弹窗实时进度、端口和日志清晰可见的轻量代理节点控制台。它的核心是让用户清楚知道节点在哪里、状态是什么、端口是什么、操作是否正在执行、最终成功或失败了多少。
