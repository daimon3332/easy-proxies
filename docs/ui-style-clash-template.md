# Clash Template UI 风格说明

## 整体风格

Clash Template 整体是浅色蓝绿系工具站风格，视觉气质偏清爽、轻量、现代。它不强调厚重后台感，而是强调快速使用、低干扰和配置编辑效率。

页面以浅色背景为基础，不使用纯灰或深色大面积铺底，而是通过非常淡的蓝绿色渐变营造空间层次。背景中带有轻微的径向光晕，左上和右上分别使用低透明度的青色、蓝色扩散，整体看起来干净、柔和，有一点科技感，但不会显得花哨。

主色使用青绿色到天空蓝的渐变体系。青绿色负责主要操作和品牌识别，蓝色用于辅助强调和次级主按钮。整体色彩饱和度适中，不是高对比霓虹风，而是偏柔和、明亮、轻盈的 SaaS 工具感。

布局上采用全宽工作台式排版，避免内容被挤在页面中央。页面横向空间被充分利用，尤其在桌面端形成明显的双栏结构：左侧作为输入和操作区域，右侧作为大面积预览或编辑区域。整体阅读路径清晰，从左到右完成操作闭环。

整体风格关键词：浅色基底、蓝绿渐变、清爽工具站、全宽工作台、左右双栏、轻量卡片、柔和圆角、低透明玻璃感、淡色边框、柔和阴影、高可读编辑区、等宽字体、低干扰操作流、现代 SaaS 感、配置工具感。

## 总体排版

Clash Template 的总体排版是“全宽工作台”而不是“居中窄容器”。页面主体应尽量占满浏览器可用宽度，只保留舒适的外边距。

桌面端推荐形成左右双栏：

- 左侧是输入区、表单区和操作区。
- 右侧是输出预览区、YAML 编辑区或代码查看区。
- 左侧负责配置输入，右侧负责结果确认。
- 横向空间优先给 URL、YAML、URI 等长文本内容。

这种排版避免了传统居中表单的问题：输入区太窄、输出区太小、宽屏空间浪费、多个卡片上下堆叠导致操作路径变长。

整体密度介于“展示型宽松”和“后台型密集”之间。控件间距紧凑但不拥挤，卡片内部有明确分组，标题、说明、输入区、按钮的层级清楚。

## 顶部导航

顶部导航采用扁平工具栏设计，类似在线工具合集站点。

导航栏不占用过多垂直空间，减少大标题和装饰性内容，把主要空间留给实际操作区域。它应该承担全局页面切换功能，而不是变成巨大的品牌展示区。

顶部导航的视觉特征：

- 横向排列。
- 高度紧凑。
- 背景为白色或半透明白色。
- 底部使用淡色分隔线。
- 当前页面使用蓝绿渐变或浅蓝绿高亮。
- 非当前项保持轻量文字按钮状态。
- 不保留过大的 Hero 标题区。
- 不保留无关按钮，例如不必要的退出按钮。

导航结构可以理解为：品牌区在左，页面入口在右或中间，当前页面有明确高亮。

## Logo 与 Favicon

Logo 是品牌视觉锚点，风格应与页面主色保持一致。

推荐 Logo 形式：

- 圆角方形。
- 蓝绿渐变背景。
- 白色大写字母，例如 C。
- 尺寸适中，不喧宾夺主。
- 与顶部导航高度协调。
- Favicon 与顶部 Logo 使用同一套视觉语言。

Logo 的作用不是制造复杂品牌图形，而是让工具站在浏览器标签页和顶部导航中有清晰识别度。

## 卡片风格

卡片设计使用圆角、浅边框、半透明白底和柔和阴影。

卡片不是完全实心白，而是带一点玻璃拟态的透明感，与背景光晕融合。边框使用淡蓝灰色，阴影很轻，只负责分层，不制造压迫感。

卡片视觉特征：

- 白色或半透明白色背景。
- 淡蓝灰色细边框。
- 圆角偏大。
- 阴影轻柔、扩散、低透明度。
- 内部留白清楚。
- 卡片头部和内容区层级明确。

卡片用于区分操作模块，但不应该过度拆分。强相关内容应该放在同一卡片或同一工作区里，避免页面被很多小卡片切碎。

## 圆角体系

圆角是 Clash Template 的重要视觉特征。

按钮、输入框、卡片、弹窗、标签、Logo 都保持较大的圆角半径，使界面更柔和、亲和，也符合浅蓝绿色系的轻量风格。

圆角体系应该统一：

- Logo 圆角较大。
- 卡片圆角较大。
- 按钮和输入框圆角中等。
- 标签和状态 Pill 使用胶囊圆角。

整体避免直角、硬边、厚边框带来的传统后台感。

## 按钮风格

按钮分层清晰，颜色承担操作语义。

按钮类型：

- 主操作按钮：青绿色渐变，用于最核心动作。
- 次主操作按钮：蓝色渐变，用于下载、转换等辅助主操作。
- 普通操作按钮：浅蓝绿色背景或白底浅边框，用于上传、清空、添加等轻量操作。
- 危险操作按钮：浅红色弱强调，用于删除、清空等风险动作。

按钮整体偏饱满，圆角明显，文字较粗，保证在浅色界面中有足够识别度。

按钮不应该过大，也不应该像传统后台按钮那样生硬。Hover 状态可以使用轻微上浮、边框变色或背景加深，但不需要夸张动画。

## 输入区与表单风格

输入区承担大量长文本输入，因此必须宽、清楚、低干扰。

输入框风格：

- 白色或接近白色背景。
- 淡蓝灰边框。
- 圆角明显。
- 聚焦时出现浅蓝绿色外发光。
- 标签文字偏小但清楚。
- 说明文字使用灰蓝色，降低干扰。

对于 URL 输入行，URL 输入框应该占主要宽度，前缀、名称、删除按钮只占必要宽度。不要让订阅链接输入框被压缩成很短的一段。

## 编辑区与代码视觉

输入区和编辑区采用代码编辑器式视觉。

文本域和 YAML 编辑区应使用等宽字体，背景接近白色或极浅蓝白，边框柔和，聚焦时出现浅蓝绿色外发光。这样能让 YAML、URI、配置文本更容易阅读，也让页面更像专业配置工具。

右侧输出区应是页面视觉重点：

- 面积大。
- 高度接近可视窗口。
- 支持长文本阅读。
- 顶部可以显示字符数、复制、下载等轻量工具。
- 视觉上接近代码编辑器或配置编辑器。

## 双栏工作台

双栏工作台是 Clash Template 的核心排版语言。

左侧输入区包含多个配置模块，右侧输出区展示生成结果或可编辑 YAML。左右区域之间应保持清晰边界，但不要使用厚重分割线。

推荐视觉关系：

- 左侧卡片组用于收集输入。
- 右侧大卡片用于结果确认。
- 右侧高度更高，强调结果区域的重要性。
- 操作按钮靠近输入流程底部或卡片头部，用户不需要来回寻找。

## 弹窗风格

弹窗用于模板编辑、长配置编辑或重要确认。

弹窗视觉特征：

- 白色或半透明白色主体。
- 圆角较大。
- 蓝灰色半透明遮罩。
- 轻柔阴影。
- 大弹窗应占据窗口大部分区域。
- 长 YAML 编辑框应占据弹窗主体空间。
- 底部或右上放保存、取消、关闭等操作。

模板编辑弹窗不应太小，因为 YAML 内容通常较长。弹窗应该让用户看到完整结构，而不是只显示局部。

## 标签、状态与提示

标签和状态使用胶囊形 Pill。

视觉特征：

- 圆角为胶囊形。
- 背景使用浅蓝、浅绿、浅红等低饱和色。
- 文字颜色比背景深，保证可读性。
- 标签用于显示状态、类型、当前模式等信息。

提示信息应低干扰，优先使用浅色说明文字。错误和危险提示才使用红色，但也以浅红弱强调为主。

## 响应式风格

桌面端以全宽双栏为主。

窄屏时：

- 顶部导航允许横向滚动或换行。
- 双栏改为上下布局。
- 输入区在上，输出区在下。
- URL 输入行可以变成纵向排列。
- 按钮组允许换行。

响应式目标不是做复杂移动端 App，而是保证工具在小屏幕上仍然能完成输入、编辑、复制。

## 应避免的设计

- 不要使用深色后台风。
- 不要使用大面积纯灰背景。
- 不要把主体内容限制在狭窄居中容器中。
- 不要使用厚重阴影和强对比边框。
- 不要把多个强相关模块拆成很多小卡片。
- 不要让输入框特别窄，尤其是 URL 和 YAML。
- 不要使用过大的标题区挤压工作台空间。
- 不要让主要操作按钮远离用户当前操作区域。

## 最终风格总结

Clash Template 的 UI 是一个浅色蓝绿系、全宽双栏工作台、轻玻璃卡片、柔和圆角、代码编辑器式预览的现代配置工具站界面。它的核心不是展示感，而是低干扰、高效率、易编辑、易确认。
