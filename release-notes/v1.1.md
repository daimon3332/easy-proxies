# v1.1 — Faster WebUI Navigation / 更快的 WebUI 页面切换

> Easy Proxies v1.1 focuses on responsive page navigation and efficient handling of large node collections.
>
> Easy Proxies v1.1 重点优化页面切换速度，以及大量节点场景下的数据加载效率。

---

## ⚡ Performance / 性能优化

- Added compact UI APIs for summaries, paginated node lists, port previews, and pool ordering.
- Replaced multi-megabyte full-node responses with small, purpose-built WebUI responses.
- Added server-side pagination, filtering, and sorting with a maximum page size of 120 nodes.
- Limited the home-page port preview to avoid loading the complete pool during navigation.
- 新增摘要、分页节点列表、端口预览和节点池排序等轻量 UI 接口。
- 使用面向 WebUI 的精简响应替代体积巨大的完整节点响应。
- 增加服务端分页、筛选和排序，单页最多返回 120 个节点。
- 限制首页端口预览数量，切换页面时不再加载整个节点池。

---

## 🧭 Navigation and Reliability / 页面切换与稳定性

- Added request cancellation so rapid tab changes stop obsolete requests.
- Added request-version checks so stale responses cannot overwrite the latest page.
- Loaded independent page data concurrently where appropriate.
- Kept missing latency, country, and port values at the end of sorted lists in both directions.
- 快速切换 TAB 时会取消已经过期的请求。
- 增加请求版本校验，旧响应不会覆盖用户最后选择的页面。
- 对互不依赖的页面数据进行并行加载。
- 无延迟、无国家和无端口的数据在升序、降序中始终排在列表末尾。

---

## 🔒 Data Exposure Reduction / 数据暴露控制

- Compact node responses omit proxy URIs and import-source details.
- Long display names and error messages are truncated before being sent to the browser.
- Batch operations resolve selected scopes on the server instead of sending every node ID from the browser.
- 精简节点响应不包含代理 URI 和导入来源详情。
- 节点名称和错误信息在返回浏览器前进行长度限制。
- 批量操作由服务端解析候选、节点池和失败范围，不再由浏览器提交全部节点 ID。
