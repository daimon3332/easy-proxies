# Easy Proxies v1.5

## English

- Fixed Clash YAML and other content imports showing a completed `0/0` task when matching nodes already existed in a failed state.
- Made URL and content imports replace the selected Tag as a fresh node snapshot.
- Prevented old failed-node state, latency, ports, and pool membership from leaking into a new import.
- Preserved import mode, format, Tag, and source metadata on testing jobs.
- Rejected empty testing commits with an explicit error instead of creating a fake completed task.
- Removed old subscription URLs associated with a replaced Tag after a content import completes successfully.
- Added regression coverage for failed-node content reimports and empty commit selections.

## 简体中文

- 修复 Clash YAML 等内容导入在已有节点处于失败状态时显示“已完成 0/0”的问题。
- URL 导入和内容导入统一按选定 Tag 替换为全新的节点快照。
- 防止旧节点的失败状态、延迟、端口和节点池状态污染新的导入任务。
- 为测速任务保留导入模式、格式、Tag 和来源等完整元数据。
- 空测速提交现在返回明确错误，不再创建虚假的 0/0 完成任务。
- 内容导入成功后，清理与被替换 Tag 关联的旧订阅 URL。
- 增加失败节点重新导入和空提交场景的回归测试。
